# 国彩真空客户数字化管理方案

## 1. 客户现状与需求

### 项目概览

| 项目 | 内容 |
|------|------|
| **客户名称** | 温州市国彩真空科技有限公司 |
| **项目名称** | 客户数字化管理平台（一期） |
| **项目类型** | 新建 |
| **核心目标** | 建立统一的客户和订单管理体系，实现老板销售可视化，沉淀客户资产 |
| **预计周期** | 4-6周（不含数据迁移和试运行） |

### 客户概况

国彩真空在温州做真空镀膜设备，主打卷对卷磁控溅射镀膜机、PVD离子镀膜机等非标定制设备，客户分布在包装、五金、眼镜镜片、电子元器件等行业。B2B 销售，订单基本都是定制，销售链条长、节点多。目前靠 Excel 和微信管客户，销售手里的客户信息带不走，老板看不到全局。

### 当前现状与挑战

| # | 挑战 | 影响 | 紧迫性 |
|---|------|------|--------|
| 1 | 客户信息分散在销售个人手机中，离职即流失 | 客户资产零沉淀，复购和转介绍无据可查 | 高 |
| 2 | 订单靠微信/电话跟进，交期状态不透明，客户体验差 | 客户反复催单，销售反复回复，效率低且易出错 | 高 |
| 3 | 非标设备排产靠Excel或口头安排，生产进度不透明 | 交期承诺难兑现，交付准时率低 | 高 |
| 4 | 设备出厂质检记录不完整，售后追溯困难 | 客诉处理无据可依，维修效率低，影响口碑 | 中 |
| 5 | 非标设备成本难精确核算，报价依赖经验 | 报价高了丢单、低了亏钱，利润空间不透明 | 中 |

老板最核心的需求其实就是四个字：看得见。看见所有客户、看见每单进度、看见团队在干什么——现有工具做不到。

## 2. 解决方案

### 整体思路

方案的核心是六个字：轻量切入，快速见效。

通用型 CRM 对这类小企业来说太重了，Excel 又太散。本方案从销售和老板的真实场景出发，用客户管理沉淀数据，用商机管理规范流程，用订单管理打通交期，用老板看板建立信任。一期聚焦 4 个模块，4-6 周上线，老板打开手机就能看到全部客户、所有订单、团队业绩。

### 功能设计

一期交付以下四个核心模块：

| 功能 | 解决的问题 | 业务价值 |
|------|-----------|---------|
| 客户管理 | 客户信息分散在销售手机里，离职即流失 | 100%客户档案入库，永久沉淀，支持公海池防止撞单 |
| 商机管理 | 商机跟进靠记忆，转化率低，阶段不透明 | 标准销售漏斗，阶段转化率自动统计，超期未跟进自动预警 |
| 订单管理 | 订单靠微信跟进，交期状态客户不知道，反复催单 | 订单全流程可视化，关键节点自动通知，客户体验提升 |
| 老板销售看板 | 老板看不到全局，不清楚团队在干什么 | 手机端随时查看：签约金额、回款进度、商机Pipeline、订单交付情况 |

**客户管理模块**

- 统一客户档案库：客户基本信息、联系人架构、历史交易、沟通记录全部入库，支持多联系人管理和权限隔离（销售只能看自己负责的客户，老板看全部）
- 公海池机制：新客户自动进入公海，分配给对应销售后专属跟进，避免撞单和抢单
- 客户合并与查重：自动识别重复客户，防止同一客户被多个销售重复录入
- 流失预警：当销售长期未跟进高价值客户时，系统自动提醒重新分配

**商机管理模块**

- 自定义销售阶段：支持配置非标设备的专属阶段，如"需求确认→方案评估→技术对接→商务谈判→签约"
- 商机卡片：每个商机关联客户、负责人、金额、预计签约日期、当前阶段，支持上传附件（技术方案、报价单草稿等）
- 转化漏斗视图：老板一眼看到每个阶段的商机数量和金额分布，识别瓶颈环节
- 超期预警：商机超过设定天数未推进，自动标记并通知负责人

**订单管理模块**

- 订单全流程状态机：从"签约→生产排期→生产中→质检→发货→安装调试→验收"各状态定义清晰，支持自定义节点（如"技术方案确认"）
- 非标设备特殊字段：技术规格、定制要求、验收标准等字段，支持附件上传
- 客户查看端：客户可通过链接查看订单当前状态，减少销售答复催单的工作量
- 关键节点通知：状态变更时自动通知相关销售和老板

**老板销售看板**

- 核心指标卡：本月/本季度签约金额、回款金额、应收账款总额
- 商机Pipeline：按阶段分组的商机数量和金额，实时更新
- 订单交付进度：按状态分组的订单数量和交付预警
- 销售排名：各销售签约金额排名（不强制展示姓名，可按编号，保护销售自尊心）
- 移动端适配：老板在手机上即可查看所有数据，无需打开电脑

### 技术方案

- 部署方式：SaaS云端部署，无需本地服务器，开箱即用
- 初始规模：支持5-20个用户，按需扩容
- 移动端：微信小程序或H5页面，老板和销售均可手机端操作
- 数据迁移：支持从Excel批量导入客户和订单数据，具体根据实际数据量评估工时
- 安全：角色权限分级（老板、销售助理），数据加密传输和存储

### 差异化优势

和通用 CRM、ERP 自带模块比，本方案的区别在这几点：

| 对比维度 | 通用型CRM（纷享销客等） | ERP自带CRM（用友/金蝶） | 本方案 |
|----------|------------------------|------------------------|--------|
| 实施周期 | 3-6个月 | 6-12个月 | 4-6周 |
| 上手难度 | 高，需要专门培训 | 中等 | 低，无需培训 |
| 非标订单适配 | 差，需大量定制 | 弱 | 专为非标设备设计 |
| 老板看板 | 有但通用，不直观 | 有但数据不全 | 直击老板核心诉求 |
| 初期投入 | 高（年费+实施费） | 极高（含ERP整体） | 可控SaaS订阅 |

## 假设说明

1. **预算**：方案按轻量SaaS订阅模式设计，假设一期预算区间为5-15万元，具体根据最终模块范围和数据迁移量调整。
2. **数据迁移**：假设现有Excel数据量中等（客户不超过500条，订单不超过200条），可在一周内完成迁移。
3. **部署**：假设客户接受SaaS部署，无需本地服务器。如需私有化部署，成本和周期将相应增加。
4. **上线时间**：假设客户无特殊时间限制，4-6周为标准实施周期。
5. **团队规模**：假设销售2-5人、销售助理1-2人，权限设计和培训方案按此规模配置。
