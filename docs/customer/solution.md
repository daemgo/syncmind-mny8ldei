## 三、解决方案架构

### 3.1 整体方案

**轻量CRM + 订单管理 + 老板看板**，以"数据沉淀 → 流程规范 → 可视决策"为主线，分期实施。

```
┌─────────────────────────────────────────────────────┐
│                  老板销售驾驶舱                        │
│           (移动端看板 · 指标预警 · 趋势分析)             │
├──────────┬──────────┬──────────┬─────────────────────┤
│ 客户管理  │ 商机管理  │ 订单管理  │     销售看板         │
│ ·档案库   │ ·阶段推进 │ ·全流程   │ ·签约金额            │
│ ·公海池   │ ·转化统计 │ ·非标字段  │ ·回款进度            │
│ ·分配回收 │ ·超期预警 │ ·状态追踪  │ ·商机Pipeline       │
│ ·沟通记录 │ ·赢单分析 │ ·交付节点  │ ·客户Top榜           │
├──────────┴──────────┴──────────┴─────────────────────┤
│              底座：用户权限 · 消息通知 · 数据导入导出       │
└─────────────────────────────────────────────────────┘

二期扩展：报价管理 → 合同管理 → 回款跟踪
三期扩展：售后维保工单 → 生产进度对接 → 成本核算
```

### 3.2 核心功能模块

#### 模块一：客户管理（REQ-001）

- **客户档案库**：基本信息（名称、行业、规模）、联系人（多角色）、历史交易、沟通记录
- **客户公海池**：未分配客户集中管理，自动回收超期未跟进客户
- **客户分配与回收**：管理员分配客户给销售，设定回收规则（如30天未跟进）
- **客户标签**：行业、规模、来源渠道（1688/展会/转介绍）、设备类型等自定义标签
- **数据导入**：支持Excel批量导入历史客户数据

**解决痛点**：防止销售离职导致客户流失，老板随时查看客户资产总量。

#### 模块二：商机管理（REQ-002）

- **标准销售流程**：初次接触 → 需求确认 → 方案报价 → 技术评审 → 商务谈判 → 签约/输单
- **阶段推进**：销售手动推进阶段，记录每个阶段的关键动作和产出
- **转化率统计**：按阶段统计转化率，识别瓶颈环节
- **超期预警**：商机在某阶段停留超过设定天数，自动提醒销售和老板
- **赢单/输单分析**：记录赢单原因和输单原因，持续优化销售策略

**解决痛点**：让每个商机可追踪、可预测，老板能看到未来Pipeline。

#### 模块三：订单全流程管理（REQ-003）

- **非标设备订单**：支持自定义字段（技术参数、定制要求、特殊工艺、交付节点）
- **订单状态追踪**：待确认 → 已签约 → 设计中 → 生产中 → 质检中 → 已发货 → 已验收
- **关键节点通知**：每个状态变更自动通知相关人员
- **订单关联**：订单关联客户、商机、合同，形成完整业务链条
- **交期预警**：距离交付节点提前N天预警，提醒跟进

**解决痛点**：解决非标设备交期难保障、进度不透明的问题。

#### 模块四：老板销售驾驶舱（REQ-004）

- **核心指标卡片**：本月签约额、本月回款额、在手订单额、应收账款总额
- **商机Pipeline**：按阶段展示商机数量和金额，漏斗图呈现
- **订单交付进度**：按期/延期/临期订单数量及占比
- **客户Top榜**：本月/季度签约客户排名
- **销售排名**：销售人员业绩排行
- **预警面板**：超期未回款、大客户流失风险、商机超期未跟进
- **移动端适配**：手机端随时查看，关键指标推送

**解决痛点**：老板不用打电话问进度，打开手机就能看到销售全貌。

### 3.3 用户角色与权限

| 角色 | 人数 | 权限范围 |
|------|------|----------|
| 老板/总经理 | 1 | 全部数据查看、审批、导出、管理配置 |
| 销售 | 2-5 | 自有客户/商机/订单的CRUD、公海池领取 |

## 四、分期实施计划

### 一期：数据沉淀 + 销售可视化（4-6周）

| 周次 | 交付内容 | 里程碑 |
|------|----------|--------|
| W1 | 需求确认、原型设计、客户档案模块 | 原型确认 |
| W2 | 商机管理模块、公海池 | 内部测试 |
| W3 | 订单全流程模块（含非标字段） | UAT测试 |
| W4 | 老板看板（核心指标+移动端） | 用户验收 |
| W5-6 | 数据导入、培训上线、陪跑 | 正式上线 |

**范围**：客户管理 + 商机管理 + 订单管理 + 销售看板
**目标**：跑通从客户录入→商机推进→订单签约→看板查看的核心流程

### 二期：商务闭环（6-8周后）

- 报价管理模块（REQ-005）
- 合同全生命周期（REQ-007）
- 回款与应收账款（REQ-006）

### 三期：价值链延伸（12周后）

- 售后维保工单管理
- 生产进度对接
- 成本核算

## 五、技术方案

### 5.1 部署方案

推荐 **SaaS云端部署**：
- 无需客户自购服务器和运维人员
- 按年订阅，初期投入低
- 数据自动备份，安全性高于本地
- 支持移动端随时访问

备选方案：私有化部署（如客户有数据安全合规要求）

### 5.2 技术选型原则

- **低学习成本**：界面简洁，3天上手，降低小型企业员工抵触
- **快速上线**：标准化模块+行业适配，4-6周可交付
- **可扩展性**：模块化架构，二期三期按需叠加
- **移动端优先**：老板和销售人员大量时间在外出差，手机端是刚需

### 5.3 数据安全

- 数据传输HTTPS加密，数据库自动备份
- 角色权限控制，销售只能查看自己的客户
- 操作日志审计，关键操作可追溯
- 支持数据导出，客户数据主权归属客户

## 六、预期价值与ROI

### 6.1 量化指标

| 指标 | 现状（推测） | 目标 | 衡量方式 |
|------|-------------|------|----------|
| 客户信息完整率 | <50% | 100% | 系统客户数 vs 实际客户数 |
| 商机转化率 | 基准值 | +20% | 签约商机数/总商机数（上线3个月） |
| 老板查看频率 | 0（无看板） | ≥3次/周 | 看板访问日志 |
| 订单按期交付率 | ~70% | ≥90% | 按期交付订单数/总订单数 |
| 回款逾期率 | 未知 | 降低50% | 逾期回款笔数/总回款笔数 |

### 6.2 隐形收益

- **客户资产沉淀**：销售离职不再丢客户，企业资产归属企业而非个人
- **决策有据可依**：老板用数据而非直觉做决策
- **团队管理提效**：销售日报/周报自动生成，减少汇报时间
- **客户体验提升**：交期更准确，跟进更及时，老客户复购率提升

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 员工数字化素养有限，使用率低 | 高 | 选择低学习成本方案，现场培训+1个月陪跑 |

| 预算不足导致项目搁置 | 高 | 提供轻量SaaS方案，按模块分期采购 |
